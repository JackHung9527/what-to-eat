# 吃什麼？🍜

一個幫你決定「今天要吃哪一家」的個人口袋名單 PWA。不用裝 app、不用註冊、不用伺服器、資料完全存在自己手機裡。

👉 **線上使用**：https://jackhung9527.github.io/what-to-eat/

---

## 為什麼做這個

每次吃飯前都在想「今天吃什麼？」，要嘛選擇障礙、要嘛一直吃同幾家。
這個小工具讓你把喜歡的店家建成口袋名單，按下按鈕隨機抽一家，點地址直接開導航就走。

---

## 功能特色

- 🍳 **五種餐別**：早餐 / 午餐 / 下午茶 / 晚餐 / 宵夜，可複選，一家店可以同時是多種餐別
- 🏷️ **分類標籤**：中式、日式、韓式、火鍋、燒烤⋯⋯等 13 種預設分類
- ⭐ **五星評分**：幫每家店打分數,再點一次同星等可取消
- 📍 **點擊導航**：地址直接連到 Apple Maps，一鍵開導航（座標也可）
- 🕐 **營業時間備註**：自由輸入格式,例如「11:00-14:00 / 週一公休」
- 📝 **個人備註**:記錄招牌菜、價位、是否要訂位等資訊
- 🎲 **隨機抽選**:依餐別 + 種類篩選後隨機抽一家,有抽獎動畫
- 📎 **智慧匯入**:貼 Google Maps 連結、或「郵遞區號+地址+店名」文字,自動拆成對應欄位
- ❌ **一鍵清除**:匯入欄內建清除按鈕,快速換下一個
- 🔍 **快速搜尋**:輸入店名後點按鈕,直接跳到 Google Maps 搜尋結果
- 📤 **匯出備份**:資料可存成 JSON 檔,防止換手機 / 清資料後遺失
- 📥 **匯入還原**:支援「合併」或「覆蓋」模式
- 🌙 **深色模式**:自動跟隨系統設定
- 📱 **PWA 支援**:加到 iPhone 主畫面後體驗接近原生 app

## 架構特色:零伺服器

**這個 app 完全沒有伺服器、沒有資料庫、沒有後端邏輯**。所有功能都在你的瀏覽器內完成。

```
你的 iPhone
    ↓
打開 https://jackhung9527.github.io/what-to-eat/
    ↓
GitHub Pages 送來一份 index.html
    ↓
手機瀏覽器執行 HTML/CSS/JavaScript
    ↓
資料存在手機 localStorage(不往外傳)
```

這種架構叫「**靜態網站 + 客戶端儲存**」或「**JAMstack**」。

### 好處

- **零營運成本**:永遠免費,不用付雲端費用
- **零維護**:沒有伺服器會當機、沒有資料庫需要備份
- **高隱私**:資料在本地,沒人看得到你的清單
- **高速度**:檔案直接從 GitHub 的 CDN 送達,載入極快
- **高穩定**:GitHub Pages 當機的機率比你 iPhone 當機還低

### 現在用到的所有外部服務

| 服務 | 用途 | 費用 |
|------|------|------|
| GitHub | 存程式碼 | 免費 |
| GitHub Pages | 網頁託管 | 免費 |
| Apple Maps | 點地址開導航 | 免費(iOS 內建) |
| Google Maps | 查店家(手動) | 免費 |

**使用者這邊不用架任何伺服器、不用付任何費用、不用管理任何帳號**(除了 GitHub)。

### 這種架構的限制

為了換到零成本零維護,犧牲了這些功能:

- 資料不能自動跨裝置同步(用匯出匯入手動解決)
- 不能讓多人共用同一份清單
- localStorage 有 5-10MB 上限(對口袋名單來說用不完)
- 無法自動抓 Google Maps 的營業時間、評分(需要付費 API)

## 怎麼使用

### 第一次打開

1. Safari 打開 https://jackhung9527.github.io/what-to-eat/
2. 點底部分享按鈕 → **加入主畫面**
3. 回到主畫面,點「吃什麼」icon 開始使用

### 建立口袋名單

1. 底部切換到 **📋 我的名單**
2. 點右上 **＋ 新增店家**
3. 填寫資訊(只有店名是必填,其他都可留空):
   - **店名**:例如「鼎泰豐」
   - **餐別**:可複選(早、午、下午茶、晚、宵夜)
   - **種類**:單選(中式、日式、速食⋯⋯)
   - **評分**:1-5 星,再點一次同星數可取消
   - **地址**:文字地址或座標,自動轉成 Apple Maps 連結
   - **營業時間**:自由文字
   - **備註**:自由文字
4. 點 **儲存**

### 隨機抽店家

1. 底部切換到 **🎲 抽店家**
2. 上方篩選條件:
   - **餐別**:全部 / 早餐 / 午餐 / 下午茶 / 晚餐 / 宵夜
   - **種類**:全部 / 特定種類
3. 點 **🎲 隨機抽一家**
4. 結果卡會顯示完整資訊,點地址即可開導航

### 編輯或刪除店家

在「我的名單」頁點任一家店,會開啟編輯視窗,可修改或刪除。

## 智慧匯入功能(新增店家加速)

在「新增店家」表單最上方有一個 **📎 從連結或地址文字匯入** 區塊,支援三種輸入方式。輸入框右側有 **× 清除按鈕**,輸入內容後自動顯示,一鍵清空方便換下一個。

### 方式 1:貼 Google Maps 連結

**支援格式**:

- `https://www.google.com/maps/place/...`
- `https://www.google.com/maps/search/...`
- `https://maps.google.com/?q=...`
- 街景模式連結(抓座標當地址)

**自動抓**:店名、地址(若連結包含)、座標

**解析邏輯細節**:
- URL 內有「郵遞區號+地址+店名」黏在一起時,自動拆解
- 優先抓 URL 裡 `!3d經度!4d緯度` 的店家精確座標
- 若沒有則退而求其次用 `@lat,lng` 的視角座標
- 抓不到文字地址時,自動用座標填入地址欄,Apple Maps 導航一樣可用

**不支援短連結**:`maps.app.goo.gl/xxx` 這種短連結純前端無法展開(瀏覽器 CORS 限制)。解法:

1. Safari 打開短連結等它跳轉
2. 從網址列複製展開後的完整長連結
3. 回到 app 貼上

### 方式 2:貼「郵遞區號+地址+店名」文字

在 Google Maps 店家頁點「複製地址」會得到這種格式:

```
104臺北市中山區南京西路12號鼎泰豐 南西店B2
```

直接貼到匯入欄,自動拆成:

- **店名**:鼎泰豐 南西店B2
- **地址**:臺北市中山區南京西路12號

支援格式變化:

- 有 / 沒有郵遞區號都行
- 含樓層「2樓」「B1」也能處理
- 「信義路二段」這種帶段號的也 OK

### 方式 3:店名旁的 🔍 按鈕

輸入店名後點 🔍 會開新分頁到 Google Maps 搜尋結果,方便你查看評分、營業時間後回來手動填。

## 資料備份(匯出匯入)

### 為什麼要備份

這個 app 的資料存在 iPhone Safari 的 localStorage,以下情況會**消失**:

- 清除 Safari 瀏覽資料
- iOS 長期不使用時自動清理 PWA 資料
- 換手機、重置 iPhone

**建議每幾週或大改清單後匯出備份一次。**

### 匯出步驟

1. **📋 我的名單** 頁 → 點 **📤 匯出**
2. 會產生檔名像 `what-to-eat_20260421.json` 的備份檔
3. iOS 跳出分享選單,可以選:
   - **儲存到檔案** → iCloud Drive(推薦,跨裝置可用)
   - **郵件** → 寄給自己當備份
   - **AirDrop** → 傳到其他 Apple 裝置
   - **訊息 / LINE** → 傳給自己或其他人

### 匯入步驟

1. **📋 我的名單** 頁 → 點 **📥 匯入**
2. 選擇之前匯出的 JSON 檔
3. 如果當前已有資料,會問你:
   - **輸入 1**:合併(保留現有 + 加入新的,店名重複會略過)
   - **輸入 2**:覆蓋(清空現有,全部改用匯入的)
   - 其他:取消

### 多裝置「偽同步」

雖然不是自動同步,但配合 iCloud Drive 可以做到類似效果:

1. iPhone 匯出 → 儲存到 iCloud Drive
2. iPad 或新手機打開 app → 匯入 → 從 iCloud Drive 選檔案
3. 每次清單有大變動,重複匯出覆蓋備份檔

## 技術細節

### 使用的技術

- 純 HTML + CSS + JavaScript(單檔)
- 無框架、無外部依賴、無建置工具
- 使用瀏覽器內建 `localStorage` 儲存資料
- 透過 PWA meta 標籤支援加入主畫面
- 匯出優先使用傳統下載,PWA 模式下自動改用 Web Share API

### 檔案結構

```
what-to-eat/
├── index.html   # 主程式(HTML + CSS + JS 全部在一個檔案)
└── README.md    # 這份說明
```

### 資料格式

每家店在 localStorage 中以以下 JSON 結構儲存:

```json
{
  "id": "lx8k2n3mk9p",
  "name": "鼎泰豐",
  "meals": ["lunch", "dinner"],
  "category": "中式",
  "stars": 5,
  "address": "台北市信義區松高路 11 號",
  "hours": "11:00-21:00",
  "note": "招牌小籠包、需排隊",
  "lat": 25.0330,
  "lng": 121.5654
}
```

**meals 欄位對應**:
- `breakfast` = 早餐
- `lunch` = 午餐
- `teatime` = 下午茶
- `dinner` = 晚餐
- `latenight` = 宵夜

**lat / lng**:若從 Google Maps 連結匯入,會自動儲存店家座標。若地址欄為「緯度,經度」格式(如 `25.0523,121.5211`),Apple Maps 也能直接用座標導航。

## 為什麼沒有「自動抓營業時間」

要自動抓營業時間必須呼叫 Google Places API,這需要:

1. 申請 Google Cloud 帳號(**必須綁信用卡**,免費額度內不會扣款但仍需綁)
2. 架一個中介伺服器保護 API Key(不能直接寫在前端)

這會破壞目前「零伺服器、零費用」的架構。為了「口袋名單」這種個人小工具,成本划不來。目前做法是讓你用 🔍 按鈕快速開到 Google Maps 查時間後手動填,效率其實差不多。

## 開發與修改

想自己改程式也很簡單:

1. Fork 或複製這個 repo
2. 編輯 `index.html`(整個 app 都在這一個檔案裡)
3. Commit 後,GitHub Pages 會在 1-2 分鐘內自動更新
4. 桌面 PWA icon 會自動抓到新版,資料不受影響

### 想加新餐別?

在 `index.html` 找到這一行:

```js
const MEAL_LABELS = { breakfast:'早餐', lunch:'午餐', teatime:'下午茶', dinner:'晚餐', latenight:'宵夜' };
```

加上你要的餐別,然後在兩處 HTML(主畫面篩選、表單選擇)加對應的 `<button class="chip">` 即可。

### 想加新種類?

找到:

```js
const CATEGORIES = ['中式','日式','韓式','西式','美式','速食','早餐店','麵食','小吃','火鍋','燒烤','飲料甜點','其他'];
```

直接改陣列即可,UI 會自動更新。
